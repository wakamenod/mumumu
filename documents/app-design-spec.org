* 1. 概要
React Native + Firebase のクイズ形式暗算・学習アプリ。
小1〜大学レベルの13段階から難易度を選択し、全7問に回答する。
上位20名がアーケードゲーム風にランキングに名前を登録できる。
最小コスト（Firebase 無料枠内）での運用と、チート完全防止を両立する。

* 2. ビジネスルール（コードだけでは分からない制約）

** 2.1 解答の判定ルール
正誤判定は「厳格な一致」が原則。自動約分・符号救済は一切行わない。

- *既約分数の義務化*: =2/4= は不正解（=1/2= が正解）
- *整数の分数表記禁止*: =3/1= は不正解（=3= が正解）
- *=-0= の禁止*: 数学的に不正な表記として不正解
- *ハッシュ比較*: ユーザー入力をそのまま SHA-256 し、サーバー提供の =answer_hash= と比較
- *サーバー再検証*: 全7問回答後、生の解答リストをサーバーに送信し Functions 側で再採点する

上記ルールのうちユーザーに関係する項目（既約分数・整数表記・=-0= 禁止）は、
トップ画面（=DifficultySelectScreen=）の「📖 解答ルール」モーダルで案内する。
クイズ画面（=NumericKeypad=）にはルール文言を表示しない。

** 2.2 入力バリデーション（NumericKeypad）
- スラッシュは文字列中に1つまで
- 先頭にスラッシュは入力不可
- スラッシュ直後にゼロは入力不可（分母ゼロ防止）

** 2.3 ユーザー名（InitialsEntryForm）
- 5文字固定（英大文字 A〜Z + ハイフン）
- 未登録状態は ="-----"= で表示

** 2.4 ランキング
- 難易度ごとに上位20名
- ソート: =correct_count= 降順 → =elapsed_time= 昇順
- 同率同タイムは =created_at= 先着順（後着は切り捨て）
- 20件未満の場合、表示時に「—」で20行に埋める

** 2.5 二重送信防止（多層ガード）
| ガード層               | 防止対象                   |
|------------------------+----------------------------|
| =gestureEnabled:false= | iOS スワイプバックによる戻り |
| =hasNavigatedToResultRef= | 二重 =router.push=      |
| =AppButton= スロットル | ボタン連打（500ms）        |
| =isSubmittingRef=      | 回答処理中の再押下         |
| =isRegistering=        | ユーザー名登録通信中の再押下 |

* 3. セキュリティ設計

** 3.1 チート対策の考え方
- クライアントには正解の *ハッシュ値のみ* を渡す（メモリスキャン対策）
- ローカル判定は UX 演出用。ランキング登録は *サーバー側再検証* が権威
- 経過時間もサーバー側で =startedAt= との差分を計算し、異常値を弾く

** 3.2 Firebase App Check
- 開発: デバッグトークン
- 本番: App Attest（iOS）/ Play Integrity（Android）

* 4. Cloud Functions API 仕様

** 4.1 getRankingFunction
指定レベルのランキング上位20件を返す。認証不要。

| リクエスト | 型       | 説明           |
|------------+----------+----------------|
| =level=    | =string= | レベル A〜M    |

レスポンス: ={ rankings: [{ rank, username, correct_count, elapsed_time }] }=
ランキング未登録時は =rankings: []= を返す（エラーにしない）。

** 4.2 submitScoreFunction
全7問の解答を送信し、サーバー側で採点・ランキング登録を行う。

| リクエスト   | 型                                    | 説明                         |
|--------------+---------------------------------------+------------------------------|
| =level=      | =string=                              | レベル A〜M                  |
| =answers=    | ={ id: string; answer: string }[]=    | 問題IDと生の解答のペア（7件） |
| =startedAt=  | =number=                              | クイズ開始時刻（Unix ms）    |

レスポンス:
| フィールド      | 型                | 説明                                      |
|-----------------+-------------------+-------------------------------------------|
| =ranked=        | =boolean=         | ランクインしたか                          |
| =rank=          | =number \vert null= | 順位。ランクイン無しは =null=            |
| =correct_count= | =number=          | 正解数                                    |
| =elapsed_time=  | =number=          | 経過時間（秒）                            |
| =claimToken=    | =string \vert null= | ユーザー名登録用 UUID v4。ランクイン無しは =null= |
| =rankings=      | =RankingEntry[]=  | ランクイン時のみ上位20件。無しは =[]= |

** 4.3 registerUsernameFunction
=claimToken= を使ってランキング行にユーザー名を書き込む。

| リクエスト   | 型       | 説明                                |
|--------------+----------+-------------------------------------|
| =level=      | =string= | レベル A〜M                         |
| =claimToken= | =string= | =submitScoreFunction= が返した UUID |
| =username=   | =string= | 5文字・英大文字+ハイフン            |

レスポンス: ={ success: true, rank: number, username: string }=
エラー時はクライアント側で固定メッセージの =Alert.alert= を表示（リトライ可能）。

* 5. Firestore 設計

=/rankings/level_{A-M}= — 難易度ごとに1ドキュメント。

=scores= 配列（最大20件）:
#+BEGIN_SRC json
{
  "username": "AAAAA",
  "correct_count": 20,
  "elapsed_time": 45.2,
  "created_at": "2026-06-13T09:27:00Z",
  "claim_token": "uuid-v4"
}
#+END_SRC

- Transaction で読み→判定→書きをアトミック実行（1 read / 1 write）
- =claim_token= は =registerUsernameFunction= 成功後に削除

* 6. データ配信設計

- 問題データは Firebase Storage に JSON で一元管理
- Storage JSON には生の =answer= を保持、Cloud Functions が SHA-256 ハッシュ化してクライアントに返す
- =question= フィールドは LaTeX 文字列（=MathDisplay= コンポーネントに渡す、デリミタ不要）
- Functions 側でメモリキャッシュ（TTL 付き）し Storage 通信量をほぼゼロに抑制

* 8. 難易度レベルの永続化

ユーザーが「スタート！」を押した時点の難易度レベルを AsyncStorage に保存し、
次回トップ画面表示時にそのレベルを初期選択状態にする。

| 項目             | 内容                                                                |
|------------------+---------------------------------------------------------------------|
| ストレージキー   | =lastSelectedLevelId=（=LAST_LEVEL_KEY= 定数で管理）                |
| 保存値           | =DifficultyId= 文字列（="M"=, ="L"=, ..., ="A"=）                  |
| 保存タイミング   | 「スタート！」ボタン押下時（=handleStart= 内）                      |
| 復元タイミング   | トップ画面（=DifficultySelectScreen=）マウント時の =useEffect=      |
| 保存値なしの場合 | =selectedIndex = 0=（小学1年生）をデフォルト表示                    |
| 無効な ID の場合 | =DIFFICULTY_LEVELS.findIndex= が =-1= → デフォルトにフォールバック |
| 読み込み失敗時   | =.catch()= でエラー吸収し、デフォルトのまま続行                     |

設計判断:
- =selectedIndex=（数値）ではなく =DifficultyId=（文字列）を保存する。
  =DIFFICULTY_LEVELS= 配列の順序変更に対して堅牢にするため。
- レベルを変更しただけでスタートしなかった場合は保存しない（意図的な選択のみ記憶）。
- ちらつき防止: =isReady= state で AsyncStorage 読み込み完了まで =opacity: 0= にする。
- 「📖 解答ルール」リンク: スタートボタン下部に配置。タップで RN =<Modal>= を開き、
  分数入力の解答ルール4項目（スラッシュ入力・既約分数・整数表記・=-0= 禁止）を表示する。
  モーダルはオーバーレイタップまたは「閉じる」ボタンで閉じる。
